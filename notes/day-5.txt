Entity Framework Core:
-------------------------------
Object Relational Mapping Model (OR/M tool)

Code First Approach

Plain Old CLR Object (POCO) type class (entity) => maps to the table
Will contain mapping information
every property has to be mapped to every column of a table

mapping:
	1. using attributes (annotations)
	2. Fluent API

DbContext => helps you map to the database
	     helps you fetch the data from database table(s) through the property (properties) all of which return(s) a collection of POCO class instances

Migration
	=> you can get the database and tables created based on the DbContext and POCO class configuration and mapping
	=> you can get the existing tables updated too
